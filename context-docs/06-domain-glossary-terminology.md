# Fi3ldMan - Domain Glossary & Terminology

## Project Terminology

| Term | Definition |
|------|-----------|
| **Fi3ldMan** | The project name for this Field Service Manual system with advanced data exploitation. The "3" in the name is a leetspeak substitution for "e" (FieldMan). |
| **Pub-5** | Publication 5 - The main regional field service manual, the original publication this repo was created for. |
| **Pub-9** | Publication 9 - Companion publication to Pub-10, currently at mockup stage. |
| **Pub-10** | Publication 10 - Specialized publication focused on spectral analysis (Grams). |
| **Grams** | Spectral/acoustic analysis data plots used in Pub-9 and Pub-10 for signal identification and diagnostic training. |
| **LOFAR** | Low Frequency Analysis and Recording - acoustic analysis technique referenced in gram content. |

## DITA Terminology

| Term | Definition |
|------|-----------|
| **DITA** | Darwin Information Typing Architecture - OASIS XML standard for structured technical documentation. |
| **Topic** | A single unit of DITA content (a .dita file), covering one subject. |
| **Map / Ditamap** | A .ditamap file that assembles topics into a publication structure. |
| **Subject Scheme** | A DITA map type that defines controlled values for profiling attributes. |
| **DITAVAL** | A filter file (.ditaval) that specifies which profiled content to include or exclude at build time. |
| **Profiling** | Conditional content processing - tagging content with attributes (audience, platform, etc.) to filter at publish time. |
| **Specialization** | Extending DITA's base element types with domain-specific elements via custom DTDs. |
| **Topicref** | A reference from a map to a topic, establishing its place in the publication hierarchy. |
| **Short description (shortdesc)** | Brief topic summary element; hidden in Fi3ldMan's published output via CSS. |
| **Related links** | Cross-references between topics, rendered as navigation aids in the output. |

## Oxygen XML Terminology

| Term | Definition |
|------|-----------|
| **Oxygen XML Editor** | The DITA authoring and publishing tool used for Fi3ldMan. |
| **WebHelp Responsive** | Oxygen's HTML5 publishing format - produces a responsive web-based help portal with search, navigation, and tiles. |
| **.opt file** | Oxygen Publishing Template - configuration file defining the template name, parameters, resources, and XSLT references. |
| **.xpr file** | Oxygen XML Project file - defines master files, profiling conditions, build scenarios. |
| **Build scenario** | An Oxygen configuration that specifies transformation type, output directory, template, and parameters. |
| **Transtype** | The output format type (e.g., `webhelp-responsive`, `pdf`). |
| **Author mode** | Oxygen's WYSIWYG-like editing mode for DITA content. |

## Equipment & Technical Domain Terms

| Term | Definition |
|------|-----------|
| **Unit** | A documented equipment installation (e.g., Unit Charlie, Unit Delta, Unit Anchors, Unit Banjo). |
| **Transducer** | A sensor/detector component (Bravo, Charlie, Delta, Echo variants). |
| **Propulsion** | The drive system of the documented equipment (diesel engines, shafts, blades). |
| **Shaft x Blade** | Notation for propulsion configuration (e.g., "2 x 4 FPP" = 2 shafts, 4 blades, Fixed Pitch Propeller). |
| **FPP** | Fixed Pitch Propeller. |
| **PTR** | A propulsion operating mode referenced for Unit Charlie. |
| **Reduction ratio** | Gearing ratio between engine and propulsion output. |
| **Harmonic** | A frequency that is an integer multiple of a fundamental frequency (H1, H2, H3, H4). |
| **SR** | Sample Rate - base frequency parameter used in the harmonics calculator. |
| **CSR** | Custom Sample Rate - secondary frequency parameter for specialized calculations. |
| **Brush Noise** | Noise generated by electrical brushes; analyzed for predictive maintenance (copper, magnesium, gold, ferrous variants). |
| **Phase G / Phase F** | Advanced signature analysis systems for propulsion-related frequencies. |
| **AAW** | Transducer pairing type referenced for Bravo Transducer. |
| **Signature** | The characteristic frequency pattern of equipment, used for identification and diagnostics. |
| **Discrete** | A specific, identifiable frequency in a signature (as opposed to broadband noise). |

## Regional Terms

| Term | Definition |
|------|-----------|
| **Region** | A geographic area with its own equipment configuration and documentation variant. |
| **Legacy** | Older generation equipment (e.g., Britain.Legacy, Spain.Legacy). |
| **Complex (Cmplx)** | Advanced equipment configurations with additional components. |
| **Composite** | Hybrid systems combining elements from multiple configurations (e.g., Wales_Composite). |
| **Wight and Man** | The Isle of Wight and Isle of Man region. |
| **Country filtering** | DITA profiling mechanism to include/exclude content by country (deny-england, deny-france, deny-spain, deny-wales). |

## Publishing & Template Terms

| Term | Definition |
|------|-----------|
| **Tile** | A clickable card on the WebHelp landing page representing a top-level topic. |
| **Protection banner** | The "COMMERCIALLY SENSITIVE" classification banner shown in header and footer. |
| **Top menu** | The hierarchical navigation menu in the page header (3 levels deep). |
| **Index terms** | Keywords assigned to topics for the publication's back-of-book style index. |
| **Context-sensitive help** | A mapping file (`context-help-map.xml`) that links help IDs to topic URLs. |
| **Page template** | HTML skeleton files (wt_index.html, wt_topic.html, etc.) that define page structure. |
| **Page fragment** | Reusable HTML components (header.xml, footer.xml) included in page templates. |

## File Extension Reference

| Extension | Purpose |
|-----------|---------|
| `.dita` | DITA topic source file |
| `.ditamap` | DITA map / publication assembly file |
| `.ditaval` | DITA conditional processing filter file |
| `.opt` | Oxygen Publishing Template configuration |
| `.xpr` | Oxygen XML Editor project file |
| `.xsl` / `.xslt` | XSLT transformation stylesheet |
| `.layout` | Oxygen XML Author mode layout configuration |
| `.css` | Cascading Style Sheets |
| `.js` | JavaScript source |
| `.bundle.js` | Bundled/compiled JavaScript module |
